Hook HealthKit - 修改微信步数

发表于 2015-12-27   |  

Hook HealthKit - 以修改微信运动读数为例

版权声明：本文为博主原创文章，未经博主允许不得转载。

看到有人把手机绑到狗狗身上来刷步数，让技术来拯救你吧。

其实博主只是想把刚刚学到的一些越狱知识实践一下而已啦。

从何开始

学逆向是本着说，逆向推动正向开发，但现在发现，其实正向也是推动逆向开发的。这次Hook HealthKit，我就先学习如何使用HealthKit，重点是如何读到读数，这样我就知道应该Hook哪里来修改。这次不是想去修改数据库信息，因为不能去修改用户真正的数据，只需要对特定的应用读数作修改，那就只需要Hook住读数据的方法后，问题就解决了。

需要解决的问题：
  1. 正向开发如何读取设备的步数
  2. 定位到微信运动读取数据的Method
  3. 如果修改读到的步数
  4. 写钩子

如何读取设备的步数

学习HealthKit的基本使用：

苹果官方文档：The HealthKit Framework 翻译

最好的当然就是Apple的Demo

读步数的Demo

所有的HealthKit读数都是通过HKHealthStore的实例发送-executeQuery:来查询结果的，不同的查询都是用HKQuery的子类来区分，步数的HKObjectType是HKQuantityTypeIdentifierStepCount

另外可以用统计集合查询HKStatisticsQuery和样本查询HKStatisticsCollectionQuery都可以查到HKQuantityTypeIdentifierStepCount的读数。

定位到微信运动读取数据的Method

创建工程

现在很明确，只需要HookHKHealthStore的-executeQuery:,看他查询的是哪个HKHealthStore就Ok了。

博主使用的是iOSOpenDev Xcode 7.2 iPhone 6 Plus 8.1 开发, 工程名HBHealthPlugin


怎么Hook HealthKit

HealthKit 是系统的Framework, 用iFunbox连接设置，找到/System/Library/Frameworks/HealthKit.framework/下的Info.plist，显然Bundle identifier为com.apple.HealthKit,工程要引入HealthKit.framework

定位Method

打开HKHealthStore.xm文件

Build一下工程， 把 HBHealthPlugin.plist 和 HBHealthPlugin.dylib拉到设备的/Library/MobileSubstrate/DynamicLibraries/

kill掉微信，再打开才会重新注入到HealthKit。

观察打出来的Log

点击进入微信运动

一看Log就很清楚了，启动应用先查询了HKObserverQuery，这是一个长时间运行的查询，它会检测HealthKit存储，并在匹配到的样本发生变化时通知你。好像和我们查的数据没什么关系；启动和进微信运动都执行了HKSourceQuery和HKStatisticsQuery，那就只能是这两个啦。

HKStatisticsQuery上面也提过，这个就可以用来查询步数的，那HKSourceQuery是什么呢，查一下资料，原来是个确定数据来源的，Health的数据来源是可以自定义的，如果读到其实APP写入或者用户自己写入的数据现展示排名那就没意义了，所以他读的数据来源只有本设备来源，那就只能是HKStatisticsQuery来读取步数了。

首先Class-dump一下头文件，找到HKStatisticsQuery.h文件，一目了然，就只有一个方法。
打开HKHealthStore.xm文件

为什么我知道completionHandler的参数？因为正向开发时，你就知道参数了。

Build一下，把HBHealthPlugin.dylib拉到设备的/Library/MobileSubstrate/DynamicLibraries/，重启微信。观察日志- - - - - - - - >>> sumQuantity： (null)，咦，这里为什么会是空的？我看健康APP里是有数据的啊，而且也跑到这里，证明读取数据就是这个方法，为什么呢。想不通的我拿着设备去买个杯咖啡压压惊先，回来重试：- - - - - - - - >>> sumQuantity： 673 count，原来我的设备放了两天，今天是没读数的，可推论他是拿今天的数据，这样我们改数据的时候就要记得对空的保护了。

修改读取到的数据

宝箱已经打开，宝物就任你拿啦。

我们知道HKQuantity就是读到的数据类，在Class-dump里找到HKQuantity的类

只有两个属性，HKUnit是数据的类型，那_value是不是就是我们要的值呢？马上试试… … - - - - - - - - >>> sumQuantity _value： 673.000000，宝物已到手。改改就好，完整代码

重新Build一下，拉文件到设备，重启微信。

Oh Yeah..


有几个点要注意
  • Xcode 7.x run工程是一定要证书，但我们钩子是没用到证书的，只要打开 /Applications/Xcode.app/Contents/Developer/Platforms/iPhoneOS.platform/Developer/SDKs/iPhoneOS9.2.sdk/SDKSettingssplit, 把DefaultProperties下的ENTITLEMENTS_REQUIRED和CODE_SIGNING_REQUIRED设为NO，就可以build过啦。PS：先把SDKSettings.plist Copy出来再改。
  • 工程要引入libsubstrate.dylib， 从iOS设备里Library/Frameworks/CydiaSubstrate.framework/CydiaSubstrate改名为libsubstrate.dylib后使用。
总结

本次主要是用正向的知识去Hook系统的Framework, 从逆向的角度，我们可以直接用Ida去解微信的包，直接看调用了哪些接口，具体是怎么实现的，不过在我要的需求里，用Ida反而更不方便，我的机子也卡爆，所以在这种简单的情景用正向思维反而更快捷。之后再用更高深的方式去解决问题吧！

Messaging（消息传递）

学习 Objective-C Runtime

小结：Objective-C 中给一个对象发送消息会经过以下几个步骤：

  1. 在对象类的 dispatch table 中尝试找到该消息。如果找到了，跳到相应的函数IMP去执行实现代码；
  2. 如果没有找到，Runtime 会发送 +resolveInstanceMethod: 或者 +resolveClassMethod: 尝试去 resolve 这个消息；
  3. 如果 resolve 方法返回 NO，Runtime 就发送 -forwardingTargetForSelector: 允许你把这个消息转发给另一个对象；
  4. 如果没有新的目标对象返回， Runtime 就会发送 -methodSignatureForSelector: 和 -forwardInvocation: 消息。你可以发送 -invokeWithTarget: 消息来手动转发消息或者发送 -doesNotRecognizeSelector: 抛出异常。
